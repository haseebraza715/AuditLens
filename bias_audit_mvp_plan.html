Week 2 — Build the agent interpretation pipeline
Apr 11 – 17  ·  Goal: LangGraph agent that reads Layer 1 JSON + task description and produces task-aware bias analysis
By end of week: user inputs a task description (e.g. "predict loan default") + uploads dataset → agent explains which bias issues matter for that task and suggests mitigations with code.
Add task description input to the API
Accept a free-text task description alongside the dataset upload. Pass both to the agent.
infra
Build LangGraph pipeline skeleton
Define nodes: parse → analyze → interpret → recommend → report. Wire state graph.
agent
Interpret node — task-aware relevance
Prompt LLM to reason: "given task X and these statistical issues, which issues are harmful and why?" Focus heavily on this prompt — it's the core value.
agent
Recommend node — mitigation strategies
Generate ranked mitigations per issue: resampling, reweighting, data collection, feature removal. Include Python code snippets.
agent
Clarifying question logic
If task description is ambiguous, agent asks 1–2 clarifying questions before interpreting. Keep simple — don't over-engineer.
agent
Basic Streamlit UI
File upload + task description text box + display JSON/text output. Just functional, not pretty yet.
infra
